# 네트워크 모니터링 프로젝트 진행 상황

## 프로젝트 개요
- **목적**: 화웨이 E8372h USB 모뎀 2~20개를 이용한 프록시/VPN 서버 구축
- **핵심 요구사항**: eno1 인터페이스의 외부 접속 우선순위 보장 및 자동 복구
- **환경**: Rocky Linux 9.6 (모든 서버 동일)

## 시스템 정보
- **OS**: Rocky Linux 9.6 (Blue Onyx)
- **IP**: 112.161.54.7/24
- **메인 인터페이스**: eno1 (MAC: 7c:10:c9:50:63:21)
- **지원 종료일**: 2032-05-31

## 핵심 요구사항
1. **eno1 우선순위**
   - 동글 연결 시에도 eno1의 메트릭이 항상 최우선
   - 외부에서 eno1 IP로 항상 접속 가능해야 함
   - 관리 트래픽(SSH 등)은 eno1 고정

2. **자동 복구**
   - eno1의 외부 접근성 정기 확인
   - 문제 발생 시 자동 복구
   - 재부팅 시 IP 할당 대기

3. **동적 환경 대응**
   - eno1 IP가 DHCP로 동적 할당
   - 인터페이스명 자동 감지 (eno1, eth0 등)
   - 동글은 항상 192.168.x.100으로 할당

## 완료된 작업 (1단계)

### 1. 프로젝트 구조
```
/opt/network-monitor/
├── scripts/
│   ├── detect_interfaces.sh    # 인터페이스 자동 감지
│   ├── monitor.sh              # 메인 모니터링 스크립트
│   ├── recovery.sh             # 복구 스크립트
│   └── health_check.js         # Node.js 웹서버
├── config/
│   └── config.conf             # 설정 파일
├── logs/
│   └── monitor.log             # 로그 파일
├── systemd/
│   ├── network-monitor.service
│   └── network-monitor-health.service
├── install.sh                  # 설치 스크립트
└── README.md                   # 문서
```

### 2. 구현된 기능
- **인터페이스 감지**: 192.168.x.100이 아닌 메인 인터페이스 자동 식별
- **외부 접근성 모니터링**: curl을 통한 실제 외부 IP 확인
- **라우팅 메트릭 관리**: 메인(100) < 동글(200+)
- **자동 복구**: 인터페이스 재시작, DNS 복구, 방화벽 확인
- **헬스체크 API**: 포트 8080에서 상태 정보 제공

### 3. API 엔드포인트
- `GET /` - 기본 상태
- `GET /status` - 상세 상태 + 로그
- `GET /interfaces` - 네트워크 인터페이스 정보
- `GET /logs` - 최근 로그
