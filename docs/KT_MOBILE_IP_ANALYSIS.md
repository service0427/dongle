# KT 모바일 IP 특성 분석

## 맞습니다! KT 모바일 IP의 특징

### 1. CGNAT (Carrier Grade NAT)
- KT는 CGNAT을 사용해 하나의 공인 IP를 수천명이 공유
- 175.223.x.x, 39.7.x.x 대역은 KT 모바일 공용 IP
- 실제로 같은 IP에서 수많은 사용자가 동시 접속

### 2. 왜 그런데도 차단되나?

#### IP가 아닌 세션 단위 감지
- 쿠팡/네이버는 IP만으로 차단하지 않음
- 대신 개별 세션의 행동 패턴을 분석
- 같은 IP라도 각 세션별로 추적

#### 세션 특징 분석
1. **TLS Fingerprint**: 각 연결의 TLS 핸드셰이크 패턴
2. **TCP Session ID**: 포트 번호와 시퀀스
3. **쿠키/세션 토큰**: 브라우저별 고유 식별자
4. **행동 패턴**: 클릭 속도, 페이지 탐색 순서

### 3. 12개 동시 연결의 문제

일반 모바일 사용자:
- 앱 1개 = 2-4개 연결
- 브라우저 탭 1개 = 6개 연결 (HTTP/2)

12개 동시 연결 패턴:
- 단일 디바이스에서는 비정상적
- 자동화 도구로 의심

### 4. 실제 차단 기준

```
같은 KT IP에서:
- 일반 사용자 A: 쿠팡 앱 사용 ✅
- 일반 사용자 B: 네이버 쇼핑 ✅  
- 봇 패턴 감지: 12개 동시 연결 ❌
```

### 5. 해결 방안

#### 1) 실제 모바일 앱처럼 동작
- 연결 수를 2-4개로 제한
- HTTP/2 사용 (다중화)
- 모바일 앱 User-Agent

#### 2) 세션 다양화
- 각 연결마다 다른 쿠키 세트
- 다른 TLS 설정
- 다른 디바이스 프로필

#### 3) 시간 분산
- 동시가 아닌 순차적 연결
- 사람처럼 랜덤 간격

### 6. 결론

KT 모바일 IP는 수천명이 공유하지만:
- IP 단위가 아닌 세션 단위로 감지
- 12개 동시 연결은 단일 디바이스로는 불가능한 패턴
- 실제 모바일 사용자처럼 행동해야 함

**권장사항**: 
- 동시 연결 4개 이하
- 각 작업 간 2-5초 대기
- 실제 앱/브라우저 에뮬레이션
